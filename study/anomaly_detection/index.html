概要

Anomaly Detection: A Survey より抜粋した内容を元に記載します。

Point Anomaly

特定のインスタンスそれ自体が異常のケースです。

ある点自体が異常時系列の場合

Contextual Anomalies

インスタンス自体は正常だが、状況によって異常とみなされるケースです。

状況異常の異常

Collective Anomalies

インスタンス単体では正常であるが、集合としてみると異常なケースです。

集合異常

ラベル情報

  • 教師あり学習: 正常と異常(複数であってもよい)のラベルが与えられるケース
  • 半教師あり学習: 正常ラベルのデータのみが与えられるケース
  • 教師なし学習: 正常と異常などが混在しているが教師ラベルが付与されていないケース

出力

  • スコア: 異常度合い(もしくは正常度合い)を表す連続的な値を返すケース
  • ラベル: 正常または異常などのラベルのみを返すケース

推定タイプ

  • 検知: 任意の時点のデータが異常かどうかを判定する。
    • Classification, Autoencoder, Metric Learning
  • 予兆検知: 任意の時点のデータが今後起こるであろう傾向を反映しているとしてとらえる。
    • 技術としては検知と同一
  • 予測: 現時点までに分かっている状態から異常が発生するかどうかを予測する。
    • Regression, RNN, LSTM

データ

  • 時系列データだが区間定義により 1 サンプルとして扱えるケース
    • 直接利用する: 単純に窓を設定してスライディングウィンドウで検出できるなら一番簡単である。
    • DTW などにより区間を動的に修正する
  • 時系列データとして扱うケース

事例紹介は、予測精度 100% というのがあるため、若干怪しい気がします。 (特定のテストデータだけとかすれば 100% が出るかもしれませんが。。。)

時系列データの解析方法には、下記の 2 種類があると主張しています。

  • Instance based classification: 時系列データを直接比較し、その距離を元に分類する。
    • 代表的な手法として、DTW + k-NN がある。
    • デメリットとして、計算量が多い。
  • Feature based classification: 時系列データから特徴量と抽出、選択し、その特徴量の距離を元に分類する。
    • 各時系列データを独立して扱うことで、計算量の抑制や取り回しが良くなる。
    • 参考文献
      • Highly comparative feature-based time-series classification
      • Distributed and parallel time series feature extraction for industrial big data applications

Deep learning: the final frontier for signal processing and time series analysis

  • Medium
  • 投稿日: 2018.11.10

信号処理と時系列データに対する deep learning が手薄いということを指摘している。 deep learning 以前の機械学習手法を説明し、その後、実際に deep learning を適用した体験について述べている。 適用対象としては、下記の通り。

  • biosignal analysis in Mawi Solutions
  • algorithmic trading

結論が最後に書かれているが、モデル作成に当たり重要な部分だけを抜粋すると下記のような感じになる。

  • シーケンシャルデータに対しては、 Autoregressive CNN > CNN > RNN の順で精度が良い
  • DTW + K-Means のような方法でクラスタリングせず、 AutoEncoder のような方法で Embedding space を利用する
  • GANs によって異常検知することもできる
  • Deep Learning 手法と従来の特徴抽出技術などを混ぜることもできる

Classical approaches

機械学習以前であれば数学的なモデルを作成して時系列データや信号処理を行っていた。

  • Time domain analysis
  • Frequency domain analysis: フーリエ変換とかウェーブレット変換すると解析できる場合があった。
  • Nearest neighboras analysis: 時系列データの場合は長さが異なる事があるため、ユークリッド距離などが使えなかった。そのため、 DTW などを利用していた。
  • (S)AR(I)MA(X) models: linear self-dependence な状況において有効な手法
  • Decomposition: 合計値などの代表値に置き換えて実施する方法
  • Nonlinear dynamics: ちょっとよくわからない
  • Machine learning: ヒューリスティックな特徴抽出を行って実現していたが、現在では deep learning による方法を利用する。

Recurrent neural nets

RNN 系は、 10 から 100 程度のステップで多変量解析の場合には有効であることが多い。 一方で、それ以上の長時間シーケンスとなると対応が難しい。 詳細は、 When Recurrent Models Don’t Need to be Recurrent に記載してある。

Convolutional neural nets

CNN は画像系で非常に成功を収めている方法である。 一方で、2D や 3D には適用するのに、なぜ 1D には適用しないのかわからないので、適用してみた。 単純に CNN の層を追加してロジスティック回帰を行うと RNN より良い結果が出ることがわかった。

Autoregressive neural nets

Clustering

DTW のような方法と k-means のような方法を組み合わせることで時系列データに対してもクラスタリングが可能である。 しかしながら、そのような方法を用いることは最適ではなく時間もかかる。 AutoEncoder のような方法を用いることで embedding space を取得することができる。 参考文献として Deep Temporal Clustering : Fully Unsupervised Learning of Time-Domain Features を提示している。

Which Anomaly Detector shold I use

  • pdf
  • 発表日: 2018.11.20

143 ページありますが、最後のまとめ部分だけ要約すると、下記の基準で選びましょうという事のようです。 一方で、どれ選ぶ?という問いに対して、異常を理解して良いアルゴリズムを選びましょうというのはどうなのでしょうか。

  • 設定パラメータが少なく、パラメータに対して頑健であること。(パラメータがないことが最も良い)
  • データセットが大きくなっても高速に動くこと。
  • 複雑度が低いこと。
  • 異なるデータ属性の下で知られた挙動をすること。
  • 様々な異常に対応できること。
  • 高次元の問題に対処できること。
  • 異常の特性を理解し、最も良いアルゴリズムを選びましょう

最初に手法の比較を開始するにあたり、色々な文献が参照されています。

  • 今回は対象としない文献
    • Distance-Based Outlier Detection: Consolidation and Renewed Bearing
    • A Comparative Evaluation of Unsupervised Anomaly Detection Algorithms for Multivariate Data
    • On the Evaluation of Unsupervised Outlier Detection: Measures, Datasets, and an Empirical Study
    • Theoretical Foundations and Algorithms for Outlier Ensembles
  • 対象とする文献
    • A Meta-Analysis of the Anomaly Detection Problem
      • isolation forest がおすすめ。
    • 書籍: Outlier Ensembles: An Introduction
      • isolation forest がおすすめ。
      • 距離ベースの手法は頑健である。
      • どの手法も、他の手法より一貫して優れた方法はなかった。
    • Defying the gravity of learning curve: a characteristic of nearest neighbour anomaly detectors
      • knn 手法を対象として調査している。
      • aNN 手法が良い。 aNN 手法は、 1NN のアンサンブル手法です。

多くの手法が参考文献として記載されています。 文中でよさそうと掛かれていて、気になったものだけ下記に抜き出します。

  • aNNE は、 isolation method に該当するとのことです。
  • 次元数が多く、巨大なデータに対してスケール可能な従来手法として下記が挙げられています。
    • REPEN: Representation learning for random distance-based outlier detection
    • 次元数が大きくなると、単純に距離を算出すると正常も異常も同程度の距離になってしまうという問題があります。 この方法に単純に対処する方法としてはコサイン類似度などを利用する方法があります。
  • Classification under Streaming Emerging New Classes: A Solution using Completely-random Trees
    • 時間経過に伴って分類対象のデータに新規のクラスが追加されるようなケースを問題としています。
    • 新規のクラスは、既存クラスの外れ値ほどは既存クラスより離れていないという仮定の下で問題を解いています。
    • 対処法として、完全にランダムな木を利用するようです。
  • Nearest-Neighbour-Induced Isolation Similarity and Its Impact on Density-Based Clustering
    • DBSCAN の距離計算を isolation forest に変えている。
    • SVM や DBSCAN の距離計算方法を isolation kernel に変えることで精度向上できる可能性が示されている。
