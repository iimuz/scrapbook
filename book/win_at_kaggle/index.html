Kaggle で勝つデータ分析の技術

評判の良い本です。

メモ

  • 時系列データ分析で、幾つか参考になりそうなコンペ情報が載っていたのでメモしておきます。
    • 来客数の将来予測: recruite restaurant visitor forecasting
    • 金融市場の将来の変数の予測: two sigma financial modeling challenge
  • 評価指標
    • Matthews Correlation Coefficient (MCC): 不均衡データに対してモデルの適性を適切に評価しやすい
    • quadratic weighted kappa: マルチクラス分類でクラス間に順序関係がある場合に利用可能。(例えば、映画のレーティングなど)
  • 特徴量の変換手法として下記が載っていました。
    • Box-cox 変換、 Jeo-Jonson 変換: Jeo-Jonson 変換の方は、 Box-Cox 変換で、さらに負の値を考慮した場合の変換となるようです。
    • RankGauss: 数値返還を順序に変換した後に、順序を保ったまま正規分布に近づける方法です。 sklearn の quontiletransfer を利用して n_quontiles を大きくし(例えば 100 とか)、distribution に normal を指定するようです。 仮に、元の分布が多峰性であっても、正規分布に近い形になるとのことです。
  • テーブルデータに対する多層パーセプトロンの参考ソリューション
    • GitHub dkivaranovic/kaggledays-recruit: Recruit Restaurant Visitor Forecasting の 5 位ソリューション。 LightGBM なども掲載されていますが、 Keras を利用した全結合層タイプのネットワークが掲載されています。
    • KaggleNotebook shixw125: 1st Place NN Model(public:0.507, private:0.513): Corporación Favorita Grocery Sales Forecasting の 1 位チームのソリューション。 Keras を利用した全結合層タイプのネットワークが掲載されています。
    • GitHub puyokw/kaggle_Otto: Otto Group Product Classification Challenge のコンペ終了後に上位者のソリューションをまとめたソリューション。 new モデルの方に、 Keras の全結合層タイプのネットワークが掲載されています。 あと、複数のモデルを組み合わせるパターンが実装されていて参考になります。
    • GitHub ChenglongChen/kaggle-HomeDepot: Home Depot Product Search Relevance のコンペで 3 位のソリューションです。 hyperopt によるパラメータチューニングが掲載されています。 keras による全結合層が使われているようです。 code/Chenglong/utils/keras_utils.py に Kears ネットワークが記載されていて、 code/Chenglong/model_param_space.py でパラメータの探索範囲が記載されています。 実際の実行におけるエントリポイントは、 [Code/Chenglong/task.pychenglongchen_task になるようです。
    • KaggleDiscussion 1st place solution: Mercari Price Suggestion Challenge で 1 位のソリューションです。 mxnet を利用してネットワークを記載しているように見えます。
  • テーブルデータに対する多層パーセプトロン以外のニューラルネット系ソリューション
    • GitHub sjvasquez/instacart-basket-prediction: Instacart Market Basket Analysis で 3 位のソリューションです。 RNN と CNN を利用しているとのことですが、 CNN が使われているところを見つけられていないです。
    • GitHub sjvasquez/web-traffic-forecasting: Web Traffic Time Series Forecasting において、 6 位のソリューションです。 wavenet ベースの手法です。 cnn.py にモデルが記載されていました。
    • GitHub ChenglongChen/tensorflow-XNN: Mercari Price Suggestion Challenge で 4 位のソリューションです。 DeepFM が使われているようです。 おそらく xnn.py がネットワークの全体像を記載しているように見えます。
