  • LinuxでMT4。インストールできない。プロキシサーバが邪魔ですよ。
    • 要は wine のバージョンが古いらしい。
  • docker の GUI を x11 forwarding するには下記のコマンド
    • ホスト側 (WSL) の設定
      • export DISPLAY = localhost:0.0
      • ここでたまに :0.0 と書いてあるサイトがあるが、きちんと localhost を付けないと失敗する。
    • クライアント側 (Ubuntu) の設定
      • /etc/ssh/sshd_config に X11Forwarding yes が必要
      • docker run --rm -it --net host -e DISPLAY=$DISPLAY -v ~/.Xauthority:/home/user/.Xauthority:rw --user $(id -u):$(id -g) fr3nd/xeyes
  • ubuntu 18.04 への wine インストール方法
    • Ubuntu 18.04にWineをインストールし、文字化け(アルファベットが豆腐)を解消する
  • wine 3.0 系だったが proxy エラーは発生します。
  • proxy 対策には https 通信用に winetricks winhttp が必要?
    • MT4 asked for proxy server during installation
    • 導入したがダメでした。
  • 公式の linux 説明 も試したが、ダメだった。
  • winehq は Release.key ではなく winehq.key に変わった??
  • wine 導入の docker image を GCP で動作させた。 動いたし X11 Forwarding は動作してきちんとローカルで操作できることまでは確認しました。 動作の遅延もあるものの、そこまでではないですし、直接操作するわけではないので、問題ないはずです。
  • wine 導入の X11 Forwarding システムでは、 MT4 を終了しないと、接続が切れませんでした。 また、接続が切れると MT4 が終了するので、放置できないことが分かりました。
  • ubuntu desktop を導入したバージョンを実行します。
  • 導入したバージョンを作成しましたが、動作させるには重いようです。 f1-micro ではメモリ不足になり、適切に動作しなかったです。
    • f1-micro 以外だと Free Instance が取得できないので、さしたるメリットがなくなります。
  • まずは、ローカルで動作させて、それ以外だと、東京リージョンで、数日だけ借りる感じにして、 挙動チェックをするのが先になりそうです。
