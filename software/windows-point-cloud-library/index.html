実行手順

vcpkg のインストール

とりあえず、 vcpkg を vendor 下に取り込み、 vcpkg のビルドを実施します。

$ git submodule add https://github.com/Microsoft/vcpkg.git vendor/vcpkg
$ cd vendor/vcpkg
$ powershell -ex bypass -f ./scripts/bootstrap.ps1

この時、 wsl を利用していると下記のようなエラーが出る場合があります。

対処法は、いったんプロジェクトフォルダをコピーして、再生成してあげるとエラーが解消します。 wsl で権限関係がうまくいっていないのだと思いますが、よくわかっていません。

$ Copy-Item point-cloud-library-sample -destination pcl-temp -recurse
$ Remove-Item -path point-cloud-library-sample -recurse -force
$ Move-Item pcl-temp point-cloud-library-sample

vcpkg を利用した point cloud library (PCL) のインストール

下記コマンドで、 pcl をインストールします。

$ cd point-cloud-library-sample/vendor/vcpkg
$ vcpkg.exe install pcl:x64-windows

上記コマンドを実行したところ依存関係も含めて 112 個のパッケージをビルドするようです。 非常に長いので気長に待つ必要があります。

結局 4 時間近くビルドするのにかかりました。 仮想環境の windows とはいえ、重いです。

あと、ビルドが終わったら vendor/vcpkg フォルダ下だけで、 32 GB も使っていました。

cmake を利用した pcl のパッケージ取得

windows でインストールしていることを前提としているパスになってしまっていますが、 下記のように記述することで PCL を読み込むことができます。

# use vcpkg
set(VCPKG_ROOT "${CMAKE_CURRENT_SOURCE_DIR}/vendor/vcpkg")
set(VCPKG_INSTALLED "${VCPKG_ROOT}/installed/x64-windows")

# use PCL
set(PCL_DIR "${VCPKG_INSTALLED}/share/pcl")
find_package(pcl 1.8.1 REQUIRED common io)

find_package で common と io を指定していますが、必要かどうかはわかっていません。

上記設定を行ってパッケージを検出した上で、下記のように include と library を指定します。

add_definitions(${PCL_DEFINITIONS})
link_directories(${PCL_LIBRARY_DIRS})
target_include_directories(hoge PRIVATE ${PCL_INCLUDE_DIRS})
target_link_libraries(hoge PRIVATE ${PCL_LIBRARIES})
