Windows で OpenSSH をドメイン環境下でインストール

2018-12-22 12:06:17 +0900 +0900 2020-01-06 00:28:11 +0000 UTC

Windows10 において、 “オプション機能の追加"から OpenSSH Server をインストールしたら失敗したので、 対処したときのメモです。

結論としては、 WSUS (Windows Server Update System) が有効になっていて、 インストールできない場合があるようなので、 一時的に WSUS を無効にしてあげることで対応できるようです。

OpenSSH Server のインストール手順

下記に手順を示します。

  1. Win + R を押して、 regedit で OK を押す。

  2. 管理者権限に昇格してレジストリエディタを開く。

  3. HKEY_LOCAL_MACHINE\SOFTWARE\Policies\Microsoft\Windows\WindowsUpdate\AU に移動する。

  4. UseWUServer の値を 0 とする。

  5. WSUS を再起動する。

    $ net stop wuauserv && net start wuauserv

  6. OpenSSH Server をインストールする。

    $ Get-WindowsCapability -Online | ? { $_.Name -like 'OpenSSH*' }
$ Add-WindowsCapability -Online -Name 'OpenSSH.Server~~~~0.0.1.0'

  7. Firewall から 22 番ポートを開放する

    $ New-NetFirewallRule -Protocol TCP -LocalPort 22 -Direction Inbound -Action Allow -DisplayName SSH

  8. C:\Windows\System32\OpenSSH に sshd_conf_default があるので、 コピーして sshd_conf にする。

    $ cd C:\Windows\System32\OpenSSH
$ cp sshd_conf_default sshd_conf

  9. OpenSSH を起動する。

    $ Restart-Service sshd
