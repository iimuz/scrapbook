概要

vim-fugitive を今までも導入していましたが、 主に diff と blame を見ることにしか利用していませんでした。 しかしながら、調べてみるとかなり便利なようなので、使い方をメモしておきます。 注意点として、 vim-fugitive は git コマンドを実行することにフォーカスしているのではなく、 現在開いているファイルに対して git コマンドを適用するイメージです。

ほとんどが公式の README.md と :help fugitive 意訳ですが、 一部コメントや他の使用方法も記載しています。

git status

git status と同様の結果を vim 上で見られるのが、 :Gstatus になります。 修正があったファイル一覧が得られるので、対象としたいファイルの行で下記のことができます。

  • c: コミット
  • p: パッチの表示
  • D: diff の表示
  • -: git add または git reset
  • =: インラインで変更点を表示または非表示
  • :Gcommit %: コミット作業

git commit

git commit と同様に :Gcommit が使えます。 そのまま、 vim 上でコメントを入力することが可能です。

git blame

git blame の出力をコードと対応付けて縦方向分割で見られるようになります。 q キーで表示を終了できます。 さらに表示された部分で下記の操作が可能です。

  • Enter: カーソル下のコミットにおける変更点を表示します。
  • o: カーソル下のコミットにおける変更点を画面を水平に分割して表示します。

git mv と git rm

それぞれ下記のような対応となっているようです。

  • Gmove: git mv
  • Gdelete: git rm

git grep

git grep と同様の動作を Ggrep で可能です。 単純な grep と異なり、 git の管理対象外のファイルを検索しません。 ただ、私の環境では既に pt で git の管理対象外のみ grep できるようにしているため、 実際にはあまり使わなさそうです。

git log

ファイルの git log を容易に見ることができます。 vim-fugitive の README.md では、 :Glog コマンドで見れると書いてありますが、実際にコマンドを実行すると下記のような警告が出ます。

:Glog will soon default to all files. Use :0Glog to target current file

したがって、コメント通り単純にファイルのログを見たいのであれば、 :0Glog を使った方がよさそうです。

git checkout と git add

:Gread で現在開いているファイルに対して git checkout -- filename と同様のことを行います。 vim-fugitive で :Gread を使った場合、ファイルの変更として行われます。 そのため、 u コマンドで修正を元に戻すことができます。

また、 :Gwrite を実行するとワークツリーとインデックスの両方に書き込みを行うため、 git add と似た動作になります。

git diff

:Gdiff で現在開いているファイルの diff を表示することができます。 この時 Gdiff rev のように revision を指定すると、 指定したリビジョンとの差分を見ることができます。

差分取り込み git add -p

:Gidff で差分表示している状態で、変更が表示されている部分の上にカーソルを置き、 dp と押すと変更部分が index に取り込まれます。 逆に do と押すと変更部分の修正を取り消すことができます。 これは、 git add -p で取り込むよりも表示が綺麗で見やすいです。 また、 ]c か [c (これは vim のショートカットです) を利用すると、 次の変更点へカーソル位置をスキップできます。
